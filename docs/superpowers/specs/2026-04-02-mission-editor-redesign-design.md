# 任务编辑器重制设计

## 概述

全新重制任务编辑器，采用组件化架构，通过三个复选框控制模块启用状态，支持拖拽放置敌人到战场预览区域。

## 需求总结

| 项目 | 决定 |
|------|------|
| 任务面板 | 始终显示（类型/名称/简报/提示/收益） |
| 模块控制 | 三个复选框（战前剧情/战斗/战后剧情） |
| 验证规则 | 不能只勾选两个剧情而不勾选战斗 → 禁止操作 |
| 布局方式 | 垂直堆叠，未启用模块收起隐藏 |
| 剧情配置 | 逐行文本输入 |
| 敌人配置 | 拖拽放置到战场预览区域 |
| 事件配置 | 单独列表，下拉框选择触发条件和位置 |

## 数据结构

复用并增强 `MissionData`：

```
MissionData (Resource)
├── 元数据
│   ├── mission_id: String
│   ├── mission_name: String
│   ├── mission_type: String (主线/支线/日常/活动)
│   ├── briefing: String (简报内容)
│   └── hint: String (提示内容)
│
├── 模块启用状态（新增）
│   ├── has_pre_battle: bool
│   ├── has_battle: bool
│   └── has_post_battle: bool
│
├── 剧情数据
│   ├── pre_battle_lines: Array[String]
│   └── post_battle_lines: Array[String]
│
├── 战斗数据
│   ├── enemy_entries: Array[{unit_id, spawn_anchor}]
│   └── event_configs: Array[{event_id, trigger_preset, spawn_anchor}]
│
└── 收益数据
    └── rewards: Array[{type, value}]
```

**变更点**：
- 新增 `has_pre_battle` / `has_battle` / `has_post_battle` 三个布尔字段，保存模块启用状态
- 其余字段保持现有结构

## 组件架构

```
scenes/mission_editor/
├── mission_editor.tscn          # 主场景，组合各组件
└── components/
    ├── task_panel.tscn          # 任务面板组件
    ├── story_editor.tscn        # 剧情编辑器组件
    ├── battle_editor.tscn       # 战斗编辑器组件
    │   ├── battlefield_preview.tscn   # 战场预览（拖拽区域）
    │   └── event_list.tscn            # 事件列表
    └── reward_editor.tscn       # 收益编辑器组件
```

**组件职责**：

| 组件 | 职责 | 输入 | 输出 |
|------|------|------|------|
| TaskPanel | 编辑任务基础信息 | MissionData | 更新元数据字段 |
| StoryEditor | 编辑剧情行（逐行文本） | lines: Array[String] | 更新剧情数组 |
| BattleEditor | 敌人拖放 + 事件配置 | enemy_entries, event_configs | 更新战斗配置 |
| RewardEditor | 编辑收益列表 | rewards | 更新收益数组 |

**复用关系**：
- `StoryEditor` 实例化两次，分别用于战前剧情和战后剧情
- `BattleEditor` 内部组合 `BattlefieldPreview` 和 `EventList`

## 交互行为

### 复选框验证逻辑

```
状态转换规则：

当前状态          用户操作           结果
─────────────────────────────────────────────
☐战前 ☐战斗 ☐战后  勾选战前          → ☑战前 ☐战斗 ☐战后 ✓
☐战前 ☐战斗 ☐战后  勾选战斗          → ☐战前 ☑战斗 ☐战后 ✓
☐战前 ☐战斗 ☐战后  勾选战后          → ☐战前 ☐战斗 ☑战后 ✓

☑战前 ☐战斗 ☐战后  勾选战后          → 禁止（无反应）
☑战前 ☐战斗 ☐战后  勾选战斗          → ☑战前 ☑战斗 ☐战后 ✓

☐战前 ☐战斗 ☑战后  勾选战前          → 禁止（无反应）
☐战前 ☐战斗 ☑战后  勾选战斗          → ☐战前 ☑战斗 ☑战后 ✓

☑战前 ☑战斗 ☐战后  勾选战后          → ☑战前 ☑战斗 ☑战后 ✓
☑战前 ☑战斗 ☑战后  取消战前          → ☐战前 ☑战斗 ☑战后 ✓
☑战前 ☑战斗 ☑战后  取消战斗          → ☐战前 ☐战斗 ☐战后 ✓
                  （自动取消两个剧情）
```

**核心规则**：
- 允许只勾选一个剧情
- 允许勾选战斗而不勾选剧情
- 禁止：两个剧情都勾选但战斗未勾选
- 取消战斗时，自动取消两个剧情（避免无效状态）

### 敌人拖拽放置

```
交互流程：

1. 敌人列表显示可用敌人类型（图标 + 名称）
2. 用户拖拽敌人图标到战场预览区
3. 战场预览区显示放置热点（6个锚点位置）
4. 放置后：
   - 敌人图标出现在对应位置
   - enemy_entries 新增一条记录 {unit_id, spawn_anchor}
5. 点击已放置敌人的 [×] 可移除
```

## 主控制器职责

```
mission_editor.gd:
├── 管理复选框状态与验证
├── 控制内容模块的显示/隐藏
├── 协调各组件与 MissionData 的数据同步
├── 处理保存/新建/返回操作
└── 提供 load_mission(mission_id) 接口
```

## 数据流

```
┌─────────────────────────────────────────────────────┐
│                  MissionData (Resource)             │
└─────────────────────────────────────────────────────┘
          ↑                    ↑                    ↑
          │ 同步               │ 同步               │ 同步
    ┌─────┴─────┐        ┌─────┴─────┐        ┌─────┴─────┐
    │ TaskPanel │        │StoryEditor│        │BattleEditor│
    └───────────┘        └───────────┘        └───────────┘
                              ↑                      ↑
                              │ 复用                  │ 组合
                        ┌─────┴─────┐          ┌─────┴─────┐
                        │ 战前/战后  │          │Battlefield│
                        └───────────┘          │ +EventList│
                                               └───────────┘
```

**同步时机**：
- 用户编辑时实时更新 MissionData
- 保存时统一持久化到 `.tres` 文件
- 加载时从 `.tres` 恢复到各组件
