# GodlingBattle 自适应 AI 行动策略设计（Phase15 扩展）

## 1. 背景与目标

当前战斗单位自动行动存在以下问题：

- 走位表现机械，缺乏局势感知。
- 某些场景下出现“看起来瞬移”或位置跳变。
- 部分单位在开局后停止有效行动，难以持续进入交战距离。
- 需要保证移动不越界、平滑、可复现。

本设计目标：

- 将战斗单位移动改造为“真正基于战场态势”的自适应 AI。
- 单位可根据敌我攻击距离、血量压力、人数态势自动：寻敌、避敌、突围、侧移。
- 移动连续平滑、无瞬移、且严格不超出战场边界。
- 保持 deterministic（同 seed 同结果）。

## 2. 范围与边界

### 2.1 In Scope

- `scripts/battle_runtime/battle_ai_system.gd` 的行动决策与移动执行逻辑。
- 运行时移动相关状态字段（例如速度/动作惯性/短期受压统计）。
- Observe 层的显示平滑（仅视觉插值，不影响 runtime 真值）。
- 回归测试：无瞬移、可接敌、不越界、突围有效。

### 2.2 Out of Scope

- 不引入网格寻路（A*）和障碍物导航系统。
- 不改变伤害结算公式与策略系统语义。
- 不改动主流程（出战前准备 -> 观战 -> 结算）。

## 3. 方案总览

采用“评分驱动 + 轻状态混合”策略（用户已确认）。

核心由四层组成：

1. 态势感知（Context Builder）
2. 动作评分（Action Scorer）
3. 轻状态混合与抗抖（Action Mixer + Hysteresis）
4. 运动执行（Velocity Integrator + Boundary Clamp）

不使用硬切换 FSM，避免状态抖动导致的观感突变。

## 4. 模块设计

### 4.1 态势感知

每 tick 对每个存活单位构建 context，包含但不限于：

- `dist_to_target`: 到当前候选目标距离
- `range_advantage`: 自身攻击距离 - 目标攻击距离
- `local_outnumbered`: 局部敌我人数差
- `recent_dps_pressure`: 最近窗口内受伤强度
- `hp_ratio`: 当前血量占比
- `boundary_risk`: 靠近边界程度

约束：

- 只能依赖 deterministic 输入（state + tick），禁止随机行为。

### 4.2 动作空间与评分

候选动作固定为：

- `approach`（接敌）
- `kite`（避敌拉扯）
- `breakout`（突围）
- `flank`（侧移/绕侧）

评分特征（方向性约束）：

- `approach_score`：距离远时升高；被围攻/高压时下降
- `kite_score`：射程优势大、短期受压高时升高；距离过远时下降
- `breakout_score`（中性触发）：受压 + 局部劣势 + 低血叠加时升高；脱离风险后快速回落
- `flank_score`：接近但未达理想攻击带时升高；靠边时下降

### 4.3 轻状态混合与抗抖

- 主动作 = 当前最高分动作
- 次动作 = 第二高分动作
- 合成方向 = `0.75 * main + 0.25 * secondary`
- 动作切换加 hysteresis：新动作分数须高于当前动作阈值才切换
- 方向与速度保留惯性，避免帧间急剧反转

### 4.4 运动执行

运行时硬约束（必须满足）：

- `|v| <= vmax`
- `|dv/dt| <= amax`
- 单 tick 位移 `<= vmax * dt`
- 更新后 position 必须 clamp 到 `arena_bounds`
- 接近边界时施加预防性回推，避免贴边抖动

说明：

- Observe 插值仅用于视觉平滑，不改变 runtime 判定位置。

## 5. 关键行为定义

### 5.1 自动寻敌

- 目标选择综合：距离、威胁值、击杀价值、对方攻击距离。
- 不采用单一最近目标，避免盲目追逐导致长时无效位移。

### 5.2 自动避敌

- 在己方劣势射程或高压状态下提高 kite 权重。
- 仍保留回接战倾向，避免“永久逃逸”。

### 5.3 自动突围（中性）

触发倾向由三要素叠加：

- 短期受伤强度高
- 局部人数劣势
- 血量进入风险区

退出条件：

- 局部压力下降后，breakout 权重快速衰减，恢复接敌/拉扯。

## 6. 防瞬移设计

### 6.1 Runtime 判定标准

定义“瞬移违规”：

- 任一实体相邻帧位置差 `delta_pos > vmax * dt * 1.2`

### 6.2 Observe 显示层

- 对 token 做插值平滑（2~3 帧趋近）
- 超大位移触发 snap（避免拖影）
- 该层不反写 runtime

## 7. 测试与验收

新增或更新测试（必须全部通过）：

1. `runtime_ai_no_teleport_test.gd`
- 断言所有单位逐帧位移不超过阈值。

2. `runtime_ai_boundary_clamp_test.gd`
- 断言任意时刻单位位置都在战场边界内。

3. `runtime_ai_engage_reachability_test.gd`
- 断言双方可进入有效交战距离，不出现长期互摸不到。

4. `runtime_ai_breakout_response_test.gd`
- 断言被围攻时出现突围倾向，脱压后恢复战斗策略。

5. `observe_motion_smoothing_visual_test.gd`
- 断言显示层位置连续，无明显跳点。

验收指标：

- 不越界：100%
- 瞬移违规：0
- 典型 seed 下无“长时空转”超时回归
- deterministic 测试保持通过

## 8. 风险与缓解

风险 1：参数耦合导致过度保守或过度激进
- 缓解：分层参数（接敌/避敌/突围）独立调优，保留黄金 seed 回归集。

风险 2：观测层平滑掩盖 runtime 问题
- 缓解：以 runtime no-teleport test 为准，插值只做表现层。

风险 3：行为抖动
- 缓解：hysteresis + 惯性 + 次优动作混合。

## 9. 实施顺序建议

1. 建立 Context + Action Scorer 数据结构（不改旧行为）
2. 接入轻状态混合器与运动执行约束
3. 接入边界回推与瞬移阈值检测
4. 接入 Observe 平滑（仅显示层）
5. 跑新增测试与关键回归

  - 全局行为风格：`均衡自适应`
  - 突围触发偏好：`中性`
  - 方案选择：`评分驱动 + 轻状态混合`
