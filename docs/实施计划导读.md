# GodlingBattle 实施计划导读

## 1. 这份文件的用途

本文件只保留“最小导航”，避免维护超长阶段索引列表造成 token 浪费。

执行进度与最新改动只看 [HANDOFF.md](./HANDOFF.md)。

## 2. 推荐阅读路径（低 token）

1. [../AGENTS.MD](../AGENTS.MD)（项目级记忆入口）
2. [HANDOFF.md](./HANDOFF.md)（当前状态 + 最新一次本次改动）
3. 仅按任务读取一个或少量 `spec/plan`

## 3. 计划入口规则

- 总计划入口：
  - [2026-03-30-godlingbattle.md](./superpowers/plans/2026-03-30-godlingbattle.md)
- 阶段计划命名模式：
  - `docs/superpowers/plans/2026-03-30-godlingbattle-phase*.md`
- 当前应跟进哪个 phase：
  - 以 [HANDOFF.md](./HANDOFF.md) 的“当前状态 / 下一阶段建议”为准

## 4. 文档维护规则

- 状态更新：只改 [HANDOFF.md](./HANDOFF.md)
- 计划步骤：只改 `docs/superpowers/plans/*.md`
- 设计定义：只改 `docs/superpowers/specs/*.md`
- 项目级记忆与协作约定：改 [../AGENTS.MD](../AGENTS.MD)

完整规则见 [文档唯一性约定.md](./文档唯一性约定.md)。

## 5. 明天接力时可直接使用的提示词

`请先阅读 AGENTS.MD 与 docs/HANDOFF.md 的“当前状态+最新一次本次改动”，然后只读取与当前任务相关的 plan/spec 继续执行。`
