# GodlingBattle 实施计划导读

## 1. 这份文件的用途

本文件只负责“从哪里开始读计划、到哪里继续执行”的导航，不维护执行状态明细。

执行进度与最新改动请统一查看 [HANDOFF.md](./HANDOFF.md)。

## 2. 阅读顺序（推荐）

1. [项目概览.md](./项目概览.md)（项目目标与边界）
2. [2026-03-30-godlingbattle-design.md](./superpowers/specs/2026-03-30-godlingbattle-design.md)（设计定义）
3. [2026-03-30-godlingbattle.md](./superpowers/plans/2026-03-30-godlingbattle.md)（总实施计划）

## 3. 分阶段计划入口

- Phase2： [2026-03-30-godlingbattle-phase2-runtime.md](./superpowers/plans/2026-03-30-godlingbattle-phase2-runtime.md)
- Phase4： [2026-03-30-godlingbattle-phase4-ui-productization.md](./superpowers/plans/2026-03-30-godlingbattle-phase4-ui-productization.md)

## 4. 文档维护规则

为避免 root 与 superpowers 文档重复维护：

- 状态更新：只改 [HANDOFF.md](./HANDOFF.md)
- 计划步骤：只改 `docs/superpowers/plans/*.md`
- 设计定义：只改 `docs/superpowers/specs/*.md`

完整规则见 [文档唯一性约定.md](./文档唯一性约定.md)。

## 5. 明天接力时可直接使用的提示词

`请先阅读 docs/HANDOFF.md、docs/项目概览.md、docs/实施计划导读.md，然后按当前阶段计划继续执行。`
