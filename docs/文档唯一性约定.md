# GodlingBattle 文档唯一性约定

## 目的

避免 `docs/` 根目录与 `docs/superpowers/` 出现同一信息多处维护，降低后续状态更新混乱，并减少新会话文档索引 token 成本。

## 唯一来源规则

1. **项目级协作记忆与低 token 阅读路径**
- 唯一来源：`AGENTS.MD`
- 用于新会话快速建立上下文，不维护逐阶段执行明细。

2. **项目全局进度、最新改动、下一步建议**
- 唯一来源：`docs/HANDOFF.md`
- 其他文档只允许引用，不重复写完整进度列表。

3. **产品目标、边界、规则（相对稳定的信息）**
- 唯一来源：`docs/项目概览.md`
- 不在此文件维护阶段进度与提交记录。

4. **设计规格（Why/What）**
- 唯一来源：`docs/superpowers/specs/*.md`
- 设计更新只改 specs，不同步复制到 root 文档。

5. **实施任务与步骤（How）**
- 唯一来源：`docs/superpowers/plans/*.md`
- 计划文档维护任务定义与阶段细节，不承担“全局状态公告”职责。

6. **基线与性能门禁策略**
- 唯一来源：`docs/benchmark-baseline-policy.md`

7. **计划导航（轻量）**
- 唯一来源：`docs/实施计划导读.md`
- 仅保留最小入口，不维护长列表索引。

8. **历史执行归档（深度追溯）**
- 唯一来源：`docs/archive/HANDOFF-*.md`
- 仅存放历史记录，不承载当前阶段判断。

## 更新流程（必须遵守）

- 更新执行进度时：只改 `docs/HANDOFF.md`。
- 变更产品目标/边界时：只改 `docs/项目概览.md`。
- 新增或调整设计定义：只改 `docs/superpowers/specs/*.md`。
- 新增或调整阶段计划：只改 `docs/superpowers/plans/*.md`。
- 调整新会话阅读入口与项目级协作约定：只改 `AGENTS.MD`（必要时同步 `docs/实施计划导读.md`）。
- 当 `docs/HANDOFF.md` 超过可读阈值（建议 250-300 行）时：将较老 `本次改动` 段落迁移到 `docs/archive/HANDOFF-*.md` 并在主文件保留归档链接。
- superpowers 发起的 sub-agent 协作默认采用单次验收（不再执行双重审核）；如涉及高风险变更再升级审核强度。该协作约定以 `AGENTS.MD` 为准。

## 冲突处理

若多个文档描述不一致，以以下优先级为准：

1. `docs/HANDOFF.md`（全局进度与最新执行状态）
2. `AGENTS.MD`（项目级协作记忆与阅读入口）
3. `docs/superpowers/specs/*.md`（设计定义）
4. `docs/superpowers/plans/*.md`（实施细节）
5. `docs/项目概览.md` 与 `docs/实施计划导读.md`（稳定摘要与轻量导航）
6. `docs/archive/HANDOFF-*.md`（历史追溯）
